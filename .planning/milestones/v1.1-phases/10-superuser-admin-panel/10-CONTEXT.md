**Gathered:** 2026-02-26

<domain>
## Phase Boundary

Администратор может управлять платформой через защищённую админ-панель: видеть статистику (юзеры, диагностики, активность), управлять пользователями (поиск, toggle is_active/is_admin) и контентом (просмотр курсов, изменение порядка уроков). Создание/удаление контента, расширенная аналитика и логирование действий — вне скоупа.

</domain>

<decisions>
## Implementation Decisions

### Dashboard и метрики
- KPI-карточки наверху — набор метрик на усмотрение Claude (опираясь на доступные данные в DB)
- Графики: линейные area chart для роста юзеров и активности
- Период по умолчанию: 7 дней, переключатель на 14d/30d/90d
- Секция «Последние события» — компактная лента (5-10 последних: регистрации, диагностики, логины)

### Управление юзерами
- Формат: классическая таблица (avatar, email, дата, статус, действия)
- Inline-действия: toggle switch для is_active и is_admin прямо в строке, мгновенное применение
- Фильтрация и пагинация — на усмотрение Claude

### Управление контентом
- Организация курсов/уроков — на усмотрение Claude (аккордеон или двухуровневая навигация)
- Изменение порядка уроков — на усмотрение Claude (drag-and-drop или стрелки)
- Редактирование метаданных — на усмотрение Claude (только просмотр vs inline edit)
- Показывать минимальную статистику: кол-во уроков в курсе, кол-во chunks по уроку (покрытие RAG)

### Доступ и навигация
- Вход в админку: только через прямой URL /admin (без кнопки в UI)
- Sidebar для /admin — на усмотрение Claude (свой или расширение основного)
- Защита /admin для не-админов — на усмотрение Claude (редирект или 403)
- Первый админ назначается вручную через SQL в Supabase SQL Editor

### Claude's Discretion
- Набор KPI-карточек на dashboard (исходя из доступных данных в DB)
- Фильтры и пагинация в таблице юзеров
- Структура отображения контента (аккордеон vs страницы)
- Механизм изменения порядка уроков (DnD vs стрелки)
- Редактирование метаданных уроков (view-only vs inline)
- Тип sidebar для /admin
- Поведение при попытке не-админа открыть /admin

</decisions>
