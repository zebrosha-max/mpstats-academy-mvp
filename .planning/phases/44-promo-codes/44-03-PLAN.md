**Page structure:**

1. **Header:** "Промо-коды" title + "Создать промо-код" button (opens creation form modal/section).

2. **Create form (toggle-able section or dialog):**
   - Тип доступа: radio/select — PLATFORM / COURSE
   - Курс: select (shown only when COURSE selected), populated from `trpc.billing.getCourses.useQuery()`
   - Длительность (дней): preset buttons 7/14/30 + custom input field
   - Макс. активаций: number input, default 1
   - Срок действия: date input (optional) or "Бессрочный" checkbox
   - Код: text input (optional, placeholder "Авто: PROMO-XXXXX") — uppercase
   - "Создать" button calls `trpc.promo.createPromoCode.useMutation()`, on success refetch list + toast

3. **Table (per D-09):**
   Columns: Код | Тип | Длительность | Использований | Истекает | Статус | Действия

   Data from `trpc.promo.getPromoCodes.useQuery()`.

   For each row:
   - Код: `promo.code` (monospace font)
   - Тип: `promo.planType` — show "Платформа" for PLATFORM, course title for COURSE
   - Длительность: `{promo.durationDays} дн.`
   - Использований: `{promo.currentUses}/{promo.maxUses}`
   - Истекает: formatted date or "—"
   - Статус: Badge — "Активен" (green) if isActive && uses < maxUses && not expired; "Использован" (gray) if currentUses >= maxUses; "Истёк" (red) if expired; "Отключён" (gray) if !isActive
   - Действия:
     - "Деактивировать" button (if isActive) -> calls `trpc.promo.deactivatePromoCode.useMutation()`
     - "Активации" button -> toggles an inline section below the row showing activations list

4. **Activations detail (expandable per row):**
   When "Активации" clicked for a promo code, fetch `trpc.promo.getPromoActivations.useQuery({ promoCodeId })`.
   Show mini-table: Пользователь | Дата | Статус подписки.

**Styling:** Follow existing admin page patterns — use Card, Badge, Button from shadcn/ui. Table with Tailwind classes matching admin/comments or admin/users.
