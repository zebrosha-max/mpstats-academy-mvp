  <action>Human verifies the full promo code flow across admin, pricing, and profile pages.</action>
  <what-built>
    Полный функционал промо-кодов:
    1. DB модели PromoCode и PromoActivation
    2. tRPC роутер с активацией (5 проверок + транзакция)
    3. Админка: создание, таблица, деактивация, просмотр активаций
    4. /pricing: раскрывающееся поле промо-кода, auth хедер
    5. Профиль: бейдж "Промо", без кнопки отмены
  </what-built>
  <how-to-verify>
    **Шаг 1: Создать промо-код в админке**
    1. Открыть http://localhost:3000/admin/promo (под ADMIN/SUPERADMIN аккаунтом)
    2. Нажать "Создать промо-код"
    3. Выбрать тип PLATFORM, 30 дней, maxUses=5
    4. Ввести код "TEST-2026" (или оставить автогенерацию)
    5. Убедиться: код появился в таблице со статусом "Активен"

    **Шаг 2: Активировать промо-код**
    1. Открыть http://localhost:3000/pricing (в другом браузере или incognito)
    2. Убедиться: в хедере кнопка "Войти"
    3. Нажать "Есть промо-код?" — поле раскрывается
    4. Ввести код, нажать "Активировать"
    5. Убедиться: редирект на /login с ?redirect=/pricing&promo=CODE
    6. Залогиниться
    7. Убедиться: вернулся на /pricing, код подставлен автоматически
    8. Нажать "Активировать"
    9. Убедиться: toast "Промо-код активирован!", редирект на /dashboard

    **Шаг 3: Проверить профиль**
    1. Открыть http://localhost:3000/profile
    2. Убедиться: подписка отображается с бейджем "Промо"
    3. Убедиться: показано "Промо-доступ" вместо цены
    4. Убедиться: показано "Доступ до DD.MM.YYYY"
    5. Убедиться: НЕТ кнопки "Отменить подписку"

    **Шаг 4: Проверить админку**
    1. Вернуться в /admin/promo
    2. Убедиться: использований стало 1/5
    3. Нажать "Активации" — убедиться: виден пользователь и дата

    **Шаг 5: Проверить повторную активацию**
    1. На /pricing снова ввести тот же код
    2. Убедиться: ошибка "Вы уже использовали этот промо-код"
