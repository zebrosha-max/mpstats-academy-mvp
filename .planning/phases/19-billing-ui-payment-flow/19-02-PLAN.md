For public page without auth context, use a separate approach: just call it and handle UNAUTHORIZED gracefully (set `retry: false, refetchOnWindowFocus: false`).
- Fetch courses for dropdown: `trpc.learning.getCourses.useQuery()` (use existing learning router)

- **COURSE card** (left): Card with `shadow-mp-card`. Title from plan.name. Price formatted with `Intl.NumberFormat('ru-RU')` + " / мес". Description "Доступ к одному курсу на выбор". Select/dropdown with 6 courses (from learning.getCourses). Button "Оформить подписку". If user has active COURSE subscription on same course — show Badge "Ваш план" (variant success) and change button to "Управление подпиской" linking to /profile.
- **PLATFORM card** (right): Card with gradient border (`border-2 border-mp-blue-500` or use Card variant gradient). Badge "Выгодно" at top-right. Title, price, description "Полный доступ ко всем курсам". Button variant="featured". If user has PLATFORM subscription — Badge "Ваш план", button "Управление подпиской" -> /profile.
- **Payment flow on button click:**
  1. Call `trpc.billing.initiatePayment.mutateAsync({ planType, courseId })`
  2. Get `{ subscriptionId, amount, planName, description }` back
  3. Call `openPaymentWidget({ publicId: process.env.NEXT_PUBLIC_CLOUDPAYMENTS_PUBLIC_ID!, description, amount, currency: 'RUB', accountId: userId, invoiceId: subscriptionId, data: { cloudPayments: { recurrent: { interval: 'Month', period: 1 } } } })`
  4. On success (promise resolves true): show toast "Оплата принята! Подписка активируется в течение минуты." using a simple alert or a div notification (no toast library needed — use state-based message). Then redirect to /profile after 3 seconds.
  5. On fail: show error message "Оплата не прошла. Попробуйте снова."
- Use `NEXT_PUBLIC_CLOUDPAYMENTS_PUBLIC_ID` env var for widget publicId.
