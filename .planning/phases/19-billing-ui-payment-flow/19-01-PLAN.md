<task type="auto">
  <name>Task 2: Billing tRPC router with all endpoints</name>
  <files>packages/api/src/routers/billing.ts, packages/api/src/root.ts</files>
  <action>
1. **Create billing router** (`packages/api/src/routers/billing.ts`):
   Import `router`, `publicProcedure`, `protectedProcedure` from `../trpc`. Import `isFeatureEnabled` from `../utils/feature-flags`. Import `TRPCError` from `@trpc/server`. Import `z` from `zod`.

   Endpoints:

   a) `isEnabled` — publicProcedure, query. Returns `boolean` from `isFeatureEnabled('billing_enabled')`.

   b) `getPlans` — publicProcedure, query. Check billing_enabled, if false return empty array. Query `prisma.subscriptionPlan.findMany({ where: { isActive: true }, orderBy: { price: 'asc' } })`. Return plans array.

   c) `getSubscription` — protectedProcedure, query. Check billing_enabled (throw NOT_FOUND if false). Find user's active subscription: `prisma.subscription.findFirst({ where: { userId: ctx.user.id, status: { in: ['ACTIVE', 'PAST_DUE', 'CANCELLED', 'PENDING'] } }, include: { plan: true, course: true }, orderBy: { createdAt: 'desc' } })`. Return subscription or null.

   d) `initiatePayment` — protectedProcedure, mutation. Input: `{ planType: z.enum(['COURSE', 'PLATFORM']), courseId: z.string().optional() }`.
      - Check billing_enabled
      - Validate: if planType is COURSE, courseId is required (throw BAD_REQUEST if missing)
      - Validate: if planType is PLATFORM, courseId must be undefined
      - Find plan: `prisma.subscriptionPlan.findUnique({ where: { type: input.planType } })`
      - Throw NOT_FOUND if plan not found or not active
      - If COURSE, verify course exists: `prisma.course.findUnique({ where: { id: input.courseId } })`
      - Check for existing ACTIVE/PENDING subscription of same type for same course (prevent duplicates)
      - Create subscription with PENDING status: `prisma.subscription.create({ data: { userId: ctx.user.id, planId: plan.id, courseId: input.courseId ?? null, status: 'PENDING', currentPeriodStart: new Date(), currentPeriodEnd: new Date() } })`
      - Create payment with PENDING status: `prisma.payment.create({ data: { subscriptionId: subscription.id, amount: plan.price, status: 'PENDING' } })`
      - Return `{ subscriptionId: subscription.id, amount: plan.price, planName: plan.name, description: plan.name }`

   e) `getPaymentHistory` — protectedProcedure, query. Input: `{ limit: z.number().min(1).max(20).default(10) }`.
      - Check billing_enabled
      - Query: `prisma.payment.findMany({ where: { subscription: { userId: ctx.user.id }, status: { not: 'PENDING' } }, orderBy: { createdAt: 'desc' }, take: input.limit, include: { subscription: { include: { plan: true } } } })`
      - Return payments array

   f) `cancelSubscription` — protectedProcedure, mutation.
      - Check billing_enabled
      - Find active subscription: `prisma.subscription.findFirst({ where: { userId: ctx.user.id, status: 'ACTIVE' } })`
      - Throw NOT_FOUND if no active subscription
      - Import and call `cancelCloudPaymentsSubscription(subscription.id)` from `@/lib/cloudpayments/cancel-api` — BUT this is a server-only module. Since tRPC routers run server-side, import it directly. However, the path alias `@/` is for the Next.js app. Instead, inline the cancel logic OR create the helper in `packages/api/src/utils/cloudpayments.ts`.
      - **Decision:** Create the cancel logic inline in the router (3 lines of fetch) rather than importing cross-package. Use `process.env.CLOUDPAYMENTS_PUBLIC_ID` and `CLOUDPAYMENTS_API_SECRET`.
      - On CP API success: update subscription status to CANCELLED and set cancelledAt (optimistic — webhook will also arrive, but handleCancellation is idempotent)
      - On CP API failure: throw INTERNAL_SERVER_ERROR with message
      - Return `{ success: true, accessUntil: subscription.currentPeriodEnd }`

2. **Register in root router** (`packages/api/src/root.ts`):
   - Import `billingRouter` from `./routers/billing`
   - Add `billing: billingRouter` to appRouter
