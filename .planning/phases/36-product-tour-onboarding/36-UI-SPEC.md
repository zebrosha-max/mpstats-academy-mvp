## Component Contracts

### 1. TourProvider

**Location:** `components/shared/TourProvider.tsx`
**Type:** Client component (`'use client'`)
**Responsibility:** Wraps `(main)/layout.tsx` children, manages tour state and driver.js instance.

```
Props: { children: ReactNode }
State: none (driver.js manages internally)
localStorage keys:
  - tour_dashboard_completed (boolean string "true")
  - tour_learn_completed (boolean string "true")
  - tour_lesson_completed (boolean string "true")
```

**Behavior:**
- On mount, checks current pathname against tour definitions.
- If `tour_{page}_completed` is NOT "true" in localStorage, auto-starts tour after 1500ms delay.
- Exposes `startTour(page: 'dashboard' | 'learn' | 'lesson')` via React Context for HelpCircle button.
- On tour completion or skip, sets `tour_{page}_completed = "true"` in localStorage.

### 2. HelpCircleButton

**Location:** `components/shared/HelpCircleButton.tsx`
**Type:** Client component (`'use client'`)
**Placement:** In `(main)/layout.tsx` header, between spacer div and UserNav.

```
Visual:
  - lucide-react HelpCircle icon, 20px
  - 40px button (w-10 h-10), rounded-lg
  - mp-gray-400 default, mp-blue-500 hover
  - transition-colors duration-200
  - title="Повторить тур"
  - Only rendered when current page has a tour definition
```

**Behavior:**
- Reads current pathname, determines which tour applies.
- On click, calls `startTour(page)` from TourProvider context.
- Hidden on pages without tours (e.g., /profile, /admin).

## Interaction Patterns

### Auto-Start Trigger

| Property | Value | Source |
|----------|-------|--------|
| Trigger | Auto-start on first visit | D-05 (Claude's Discretion) |
| Delay | 1500ms after page mount | Allows page content to render + animate |
| Condition | `localStorage.getItem('tour_{page}_completed') !== 'true'` | D-08 |
| Frequency | Once per page (3 independent flags) | Each tour tracked separately |

### Tour Navigation

| Action | Behavior |
|--------|----------|
| "Далее" click | Advance to next step with 200ms transition |
| "Назад" click | Return to previous step |
| "Пропустить" click | Close tour, set `tour_{page}_completed = "true"` |
| "Готово" click (last step) | Close tour, set `tour_{page}_completed = "true"` |
| Overlay click | No action (tour stays open — prevent accidental dismissal) |
| Escape key | Close tour, set completed flag |
| Keyboard Tab/Enter | driver.js built-in keyboard navigation |

### Mobile Adaptation (D-07)

| Desktop Element | Mobile Replacement | Rationale |
|----------------|-------------------|-----------|
| Sidebar nav (`data-tour="sidebar-nav"`) | MobileNav (`data-tour="mobile-nav"`) | Sidebar hidden on mobile (`hidden md:flex`) |
| Popover max-width 340px | Popover max-width `calc(100vw - 32px)` | Prevent horizontal overflow on 375px screens |
| Popover positioned by driver.js | Same — driver.js auto-positions relative to viewport | Built-in responsive positioning |

**Detection:** Use `window.matchMedia('(min-width: 768px)')` at tour start to select desktop vs mobile step definitions.

### HelpCircle Button Visibility

| Page | Button Visible | Tour Available |
|------|---------------|----------------|
| /dashboard | Yes | Dashboard tour |
| /learn | Yes | Learn tour |
| /learn/[id] | Yes | Lesson tour |
| /diagnostic/* | No | No tour defined |
| /profile | No | No tour defined |
| /admin/* | No | No tour defined |

### z-index Stack

| Layer | z-index | Element |
|-------|---------|---------|
| Sidebar | z-30 | Fixed sidebar (desktop) |
| Header | z-40 | Sticky header |
| MobileNav | z-50 | Bottom mobile nav |
| driver.js overlay | z-[10000] | Tour overlay (driver.js default) |
| driver.js popover | z-[10001] | Tour popover (driver.js default) |
| CookieConsent | z-[9999] | Cookie banner (existing) |

Note: driver.js uses z-index 10000+ by default, which naturally layers above CookieConsent (z-9999). If both are visible simultaneously (unlikely but possible on first visit), the tour overlay covers the cookie banner. This is acceptable -- user completes tour first, then sees cookie banner after.
