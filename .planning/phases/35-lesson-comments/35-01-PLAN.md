1. Add LessonComment model to schema.prisma after the ChatMessage model section (before AI CACHE section):

```prisma
// ============== LESSON COMMENTS ==============

model LessonComment {
  id        String   @id @default(cuid())
  lessonId  String
  userId    String
  content   String   @db.Text
  parentId  String?
  createdAt DateTime @default(now())

  user     UserProfile    @relation(fields: [userId], references: [id], onDelete: Cascade)
  parent   LessonComment? @relation("CommentReplies", fields: [parentId], references: [id], onDelete: Cascade)
  replies  LessonComment[] @relation("CommentReplies")

  @@index([lessonId, createdAt(sort: Desc)])
  @@index([parentId])
}
```

2. Add the relation to UserProfile model — add `lessonComments LessonComment[]` to the UserProfile relations list (after `chatMessages ChatMessage[]` line).

3. Run `npx shadcn@latest add textarea` from `apps/web/` directory to install the shadcn textarea component.

4. Run `pnpm db:generate` to regenerate Prisma client.

5. Run `pnpm db:push` to push schema to Supabase.

Note: Using `@db.Text` for content to allow up to 1500 chars without varchar limit issues. Self-relation `CommentReplies` enables 1-level threading. Cascade delete on parent means deleting a root comment removes all replies (per D-09, UI-SPEC delete contract). Index on `[lessonId, createdAt DESC]` optimizes the primary query pattern.
