## Interaction Contracts

### Desktop Layout (lg: breakpoint)
- AI-chat Card height: **400px** (reduced from current 500px to leave room for comments below)
- Comments section placed directly below AI-chat in the right column (`space-y-4` between)
- Right column scrolls naturally with page (no separate scroll container)
- Both chat and comments visible on a 1080px-height viewport with moderate scroll

### Mobile Layout (below lg:)
- Comments section rendered below lesson navigation buttons (after the `border-t` nav area)
- Full width, no sidebar constraints
- Identical component, just different placement in DOM

### New Root Comment
1. User types in textarea at top of comments section
2. "Отправить" button enabled when textarea is non-empty and <= 1500 chars
3. On submit: optimistic insert at top of list (newest first, D-08), textarea cleared
4. On server error: toast error via sonner, remove optimistic comment
5. Enter submits (Ctrl+Enter on textarea for newline is NOT required -- single Enter does NOT submit in textarea)

### Reply to Comment
1. User clicks "Ответить" on a root comment
2. Inline CommentInput appears directly below the root comment's replies (within the thread)
3. autoFocus on the textarea
4. "Отмена" button cancels and hides the input
5. On submit: optimistic insert as last reply in thread (ASC order, D-08), input hidden
6. Only root comments have "Ответить" -- replies to replies are not supported (1-level nesting)

### Delete Comment
1. User clicks trash icon on own comment (or admin on any comment)
2. AlertDialog confirmation: "Удалить комментарий?" / "Это действие нельзя отменить."
3. On confirm: optimistic remove from list
4. If deleting a root comment with replies: all replies are also removed (cascade)
5. On server error: toast error, restore comment

### Pagination
1. Initial load: first 20 root comments (with all their replies expanded inline)
2. If more comments exist: "Показать ещё" outline button at bottom
3. Clicking loads next 20, appended to list
4. Button disappears when all comments loaded

### Empty State
- Centered in comments section area
- Icon + text, no action button (the input form above serves as CTA)

### Loading State
- 3 skeleton comment placeholders (avatar circle + text lines shimmer)
- Shown on initial load only, not on "Показать ещё" (that uses a spinner on the button)

## Text Format Decision

**Decision: Plain text** (Claude's discretion per D-04).

Rationale: Comments are short user messages, not technical documentation. Plain text avoids XSS surface, keeps UI simple, and avoids rendering inconsistencies. SafeMarkdown is overkill for comments. Line breaks preserved via `whitespace-pre-wrap`.
