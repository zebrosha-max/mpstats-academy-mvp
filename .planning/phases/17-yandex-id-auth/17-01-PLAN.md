  <action>
    1. Create `apps/web/src/app/api/auth/yandex/callback/route.ts` — GET handler:
       a. Read `code` and `state` from searchParams
       b. If no code → redirect to `/login?error=missing_code`
       c. Read `yandex_oauth_state` cookie, verify it matches `state` param (CSRF protection). If mismatch → redirect to `/login?error=invalid_state`
       d. Delete the state cookie
       e. Instantiate `YandexProvider`, call `exchangeCode(code)` to get accessToken
       f. Call `getUserInfo(accessToken)` to get `{ id, email, name }`
       g. Use `getSupabaseAdmin()` to find user by email: try `admin.getUserByEmail(email)`, fallback to `admin.listUsers()` filter
       h. If user not found → `admin.createUser({ email, email_confirm: true, user_metadata: { full_name: name, yandex_id: id } })`
       i. Generate session: `admin.generateLink({ type: 'magiclink', email })` → `auth.verifyOtp({ token_hash: linkData.properties.hashed_token, type: 'magiclink' })`
       j. Update UserProfile.yandexId via Prisma `upsert` (handle race condition with handle_new_user trigger)
       k. Create redirect Response to /dashboard
       l. Set Supabase session cookies on the response using createServerClient with cookie setAll writing to response.cookies
       m. Wrap everything in try/catch → on error redirect to `/login?error=auth_callback_error`

    2. Modify `apps/web/src/lib/auth/actions.ts`:
       - REMOVE `signInWithGoogle()` function entirely
       - ADD `signInWithYandex()` function:
         - Generate state via `crypto.randomUUID()`
         - Store state in httpOnly cookie `yandex_oauth_state` (maxAge: 600, secure in prod, sameSite: lax, path: /)
         - Instantiate `YandexProvider()`, get `authorizeUrl(state)`
         - Call `redirect(url)`
       - Keep all other functions (signUp, signIn, signOut, resetPasswordRequest, updatePassword, getUser, getSession) unchanged

    3. Create `apps/web/tests/auth/yandex-oauth.test.ts` testing callback route handler with mocked dependencies (YandexProvider, supabase admin, Prisma).
