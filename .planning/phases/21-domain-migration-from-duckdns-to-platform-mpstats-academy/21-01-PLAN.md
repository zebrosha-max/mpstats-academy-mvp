<task type="auto">
  <name>Task 2: Update VPS infrastructure (Nginx + SSL + env + Docker rebuild)</name>
  <files>VPS: /etc/nginx/sites-available/maal.conf, /home/deploy/maal/.env.production</files>
  <action>
Execute the following steps sequentially via SSH to VPS (deploy@89.208.106.208). All commands run on the VPS, NOT locally.

**Step 1: Verify DNS resolution from VPS**
```bash
dig +short platform.mpstats.academy
# Must return 89.208.106.208. If not, wait and retry.
```

**Step 2: Update Nginx server_name**
Edit `/etc/nginx/sites-available/maal.conf` (or whatever the MAAL Nginx config is named):
- Change `server_name academyal.duckdns.org;` to `server_name platform.mpstats.academy;`
- Keep ALL other settings unchanged (proxy_pass, proxy_buffer_size 128k, headers, etc.)
- Remove any certbot-added SSL lines referencing the old DuckDNS cert (certbot will re-add them for the new domain)

Then validate and reload:
```bash
sudo nginx -t
sudo systemctl reload nginx
```

**Step 3: Obtain SSL certificate**
```bash
sudo certbot --nginx -d platform.mpstats.academy
```
Certbot will auto-configure SSL directives in the Nginx config and reload Nginx.

**Step 4: Update .env.production**
Edit `/home/deploy/maal/.env.production`:
- Change `NEXT_PUBLIC_SITE_URL=https://academyal.duckdns.org` to `NEXT_PUBLIC_SITE_URL=https://platform.mpstats.academy`
- All other env vars remain unchanged

**Step 5: Rebuild and restart Docker container**
```bash
cd /home/deploy/maal
docker compose down
docker compose build --no-cache
docker compose up -d
```
Wait for container to become healthy (~1-2 minutes).

**Step 6: Verify health**
```bash
curl -s https://platform.mpstats.academy/api/health
```
Should return `{"status":"ok",...}` with database connected.

**Step 7: Delete old DuckDNS certificate**
```bash
sudo certbot delete --cert-name academyal.duckdns.org
```

**IMPORTANT per user decision:**
- `NEXT_PUBLIC_*` vars are baked at build time — `docker compose build --no-cache` is MANDATORY after env change
- `proxy_buffer_size 128k` MUST remain in Nginx config (required for Supabase auth cookies)
- No redirect from old domain — just disable it
- No maintenance page needed (few users, ~5 min downtime during rebuild)
