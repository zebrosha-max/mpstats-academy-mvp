**Avatar upload section** (per D-01, D-02, D-03):

1. Add a new Card at the top of profile content with title "Фото профиля":
   - Current avatar preview (96x96 circle): shows uploaded image or gradient initials fallback
   - "Загрузить фото" Button that triggers hidden `<input type="file" accept="image/jpeg,image/png,image/webp" />`
   - "Удалить" Button (ghost variant, only shown if avatar exists)
   - Helper text: "JPG, PNG или WebP, до 2 МБ"

2. Client-side resize logic (per D-01 — canvas API, no external lib needed):
   - On file select via input onChange:
     a. Validate file size <= 2MB (`file.size <= 2 * 1024 * 1024`), show `toast.error('Файл слишком большой. Максимум 2 МБ')` if exceeded
     b. Validate MIME type is image/jpeg, image/png, or image/webp
     c. Load image into HTMLImageElement via `URL.createObjectURL`
     d. Draw onto canvas at max 256x256 (maintain aspect ratio, crop to square from center)
     e. Export as webp: `canvas.toBlob(blob => ..., 'image/webp', 0.85)`
   - Show preview immediately from canvas blob (local URL)

3. Upload flow:
   ```typescript
   const supabase = createClient();
   const timestamp = Date.now();
   const path = `${userId}/${timestamp}.webp`;

   // Upload to Supabase Storage
   const { error: uploadError } = await supabase.storage
     .from('avatars')
     .upload(path, blob, { contentType: 'image/webp', upsert: false });

   if (uploadError) {
     toast.error('Ошибка загрузки', { description: uploadError.message });
     return;
   }

   // Get public URL
   const { data: urlData } = supabase.storage.from('avatars').getPublicUrl(path);

   // Save URL to profile via tRPC
   await updateProfile.mutateAsync({ avatarUrl: urlData.publicUrl });
   toast.success('Фото обновлено');
   ```

4. Delete flow:
   - Call `supabase.storage.from('avatars').remove([currentPath])` (extract path from current avatarUrl)
   - Call `updateProfile.mutate({ avatarUrl: null })`
   - Show `toast.success('Фото удалено')`

5. Get userId from profile data: `const userId = profile?.id`

6. Loading state: disable buttons and show spinner during upload/delete.

**Important:** The existing `updateProfile` mutation is already defined in this file. Reuse it — do NOT create a new mutation. Just call `updateProfile.mutate({ avatarUrl: publicUrl })` after Storage upload succeeds.

**Also important:** After successful upload or delete, call `refetch()` (already exists in this component) to refresh profile data, which will propagate to UserNav through layout re-render on next navigation.
