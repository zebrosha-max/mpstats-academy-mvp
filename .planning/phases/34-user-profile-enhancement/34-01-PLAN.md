**1. Create `scripts/sql/create_avatars_bucket.sql`** (per D-10, D-11, D-12):

SQL script to run manually in Supabase SQL Editor. Creates public bucket `avatars` with RLS policies:

```sql
-- Create avatars bucket (public — avatars are not secret, per D-12)
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'avatars',
  'avatars',
  true,
  2097152, -- 2MB limit (per D-02)
  ARRAY['image/jpeg', 'image/png', 'image/webp']
)
ON CONFLICT (id) DO NOTHING;

-- RLS policy: authenticated users can upload to their own folder (per D-10)
CREATE POLICY "Users can upload own avatar"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id = 'avatars'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- RLS policy: authenticated users can update their own files
CREATE POLICY "Users can update own avatar"
ON storage.objects FOR UPDATE
TO authenticated
USING (
  bucket_id = 'avatars'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- RLS policy: authenticated users can delete their own files
CREATE POLICY "Users can delete own avatar"
ON storage.objects FOR DELETE
TO authenticated
USING (
  bucket_id = 'avatars'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- RLS policy: anyone can read avatars (public bucket)
CREATE POLICY "Anyone can read avatars"
ON storage.objects FOR SELECT
TO public
USING (bucket_id = 'avatars');
```

b) Add `getAvatarUploadUrl` procedure — generates the storage path for the client to upload directly:

```typescript
getAvatarUploadUrl: protectedProcedure.query(async ({ ctx }) => {
  const timestamp = Date.now();
  const path = `${ctx.user.id}/${timestamp}.webp`;
  const url = `${process.env.NEXT_PUBLIC_SUPABASE_URL}/storage/v1/object/avatars/${path}`;
  return { path, url, bucket: 'avatars' };
}),
```

Note: The actual upload happens client-side using `supabase.storage.from('avatars').upload(path, file)`. This procedure just provides the path convention (per D-11: `avatars/{userId}/{timestamp}.webp`).

deleteAvatar: protectedProcedure
