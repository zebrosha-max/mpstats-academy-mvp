## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-08 | Spoofing / Access Control | /welcome route | mitigate | `welcome/layout.tsx` (Server Component) делает `getUser()` → `redirect('/login')`; `/welcome` также добавлен в `middleware.ts protectedRoutes` (defense in depth). Неавторизованный не увидит визард. |
| T-56-09 | Elevation of Privilege | (main)-layout guard bypass | accept | Гард — серверный компонент, выполняется до рендера; обойти на клиенте нельзя. `redirect()` бросает исключение, прерывает рендер. Низкий риск. |
| T-56-10 | Tampering | redirect-loop при сбое мутации | mitigate | `router.push` строго в `onSuccess` — при упавшей `complete` пользователь остаётся на `/welcome` с сохранённым стейтом, повторяет (Pitfall 4). E2E косвенно покрывает успешный путь. |
| T-56-11 | Tampering (XSS) | reframe-строка из goalText/goals | mitigate | Reframe — клиентский шаблон, рендерится как текст через React-эскейпинг, без `dangerouslySetInnerHTML`. |
