# Phase 56 — UI Design Contract

> Визуальный и интеракционный контракт фазы «Entry Flow Redesign» (онбординг-визард `/welcome` + де-гейтинг урока).
> Сгенерировано gsd-ui-researcher, проверяется gsd-ui-checker.
>
> **Дизайн-система зафиксирована.** Этот контракт переиспользует существующие токены MPSTATS Academy
> (`tailwind.config.ts`) и shadcn-компоненты из `apps/web/src/components/ui/`. Новый дизайн-язык НЕ вводится.
> Все user-facing строки — на русском.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (уже инициализирован — `apps/web/components.json`) |
| Preset | not applicable — кастомная дизайн-система MPSTATS поверх shadcn (`tailwind.config.ts`) |
| Component library | Radix UI (`@radix-ui/*`) через shadcn-обёртки |
| Icon library | `lucide-react` ^0.453.0 (новый код — lucide; в legacy-страницах встречаются inline SVG, для новых компонентов использовать lucide) |
| Font | Inter (`font-sans` / `font-heading`), system-ui fallback |

**Готовые компоненты для переиспользования** (`apps/web/src/components/ui/`):
`button`, `card`, `badge`, `textarea`, `checkbox`, `dialog`, `skeleton`, `sonner` (toast).

**Замечание о цвете прототипа.** ПРОТОТИП 03 рисует ассистента и акценты «фиолетовыми». В коде платформы
бренд-первичный цвет — **синий** `mp-blue-500 #2C4FF8`. Фиолетовый из прототипа реализуется штатным
`mp-blue` (это и есть «purple/blue-led» палитра академии). Зелёная карта развилки — `mp-green`.
Новый фиолетовый токен НЕ добавляется.

## Spacing Scale

Шкала из дизайн-системы (Tailwind 4px-base). Кратность 4 соблюдена.

| Token | Tailwind | Value | Usage в фазе 56 |
|-------|----------|-------|-----------------|
| xs | `gap-1` / `p-1` | 4px | Зазор иконка↔текст в чипе |
| sm | `gap-2` / `p-2` | 8px | Внутренние зазоры чипов, бейджей |
| md | `gap-4` / `p-4` | 16px | Зазор между чипами/опциями, padding мобильной карточки |
| lg | `gap-6` / `p-6` | 24px | Padding карточки-оболочки визарда (`CardContent` уже `p-6`) |
| xl | `gap-8` / `p-8` | 32px | Вертикальный ритм между блоками шага (заголовок ↔ контент ↔ футер) |
| 2xl | `py-12` | 48px | Внешний вертикальный отступ контента визарда от краёв экрана (desktop) |
| 3xl | `gap-16` | 64px | не используется в этой фазе |

Исключения:
- Высота интерактивных контролов задаётся `Button` size-вариантами (`h-10` / `h-12` / `h-14`) — это
  компонентный контракт, не spacing-token. Тач-таргеты карточек-опций и чипов — минимум 44px по высоте
  (`min-h-11`) на мобильном.
- Сетка карточек маркетплейсов — `gap-3` (12px) допустимо как промежуточный шаг для плотной grid-раскладки.

## Typography

Роли мапятся на готовые `fontSize`-токены из `tailwind.config.ts`. Inter, 2 веса (400 / 600).

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Display | `text-display-sm` | 36px (2.25rem) | 700 | 1.2 | Заголовок развилки «[Имя], мы готовы помочь вам расти 🚀» (desktop) |
| Heading | `text-heading-xl` | 30px (1.875rem) | 600 | 1.3 | Заголовок шага визарда («Добро пожаловать…», «Привет, [Имя]! 👋») |
| Heading-sm | `text-heading-sm` | 18px (1.125rem) | 600 | 1.5 | Заголовок карты развилки, заголовок хинт-карточки на уроке |
| Body | `text-body` | 16px (1rem) | 400 | 1.6 | Подзаголовки шагов, описания, текст в `textarea` |
| Body-sm | `text-body-sm` | 14px (0.875rem) | 400 | 1.5 | Метки чипов/опций, helper-строки, бейджи развилки |
| Caption | `text-caption` | 12px (0.75rem) | 400 | 1.4 | Privacy-нота под полем шага 1, степпер-подписи |

Мобильная адаптация заголовков: на `<sm` заголовок развилки понижается до `text-heading-xl` (30px),
заголовок шага — до `text-heading-lg` (24px). Веса не меняются. `text-heading-xl` и `text-heading-lg`
здесь — адаптивные варианты тех же ролей Display/Heading, а не дополнительные роли; число ролей
остаётся 6.

**Правило:** ровно 2 веса (`font-normal` 400 для body/caption, `font-semibold` 600 для всех заголовков
и меток-кнопок). `font-bold` (700) допустим ТОЛЬКО внутри токена `display-sm` (он несёт 700 встроенно).

## Component Inventory & Interaction Contract

### `/welcome` layout
- Standalone-роут вне `(main)`. Минимальный полноэкранный layout: фон `bg-mp-gray-50`, БЕЗ сайдбара/навигации.
- Контент центрируется: `flex min-h-screen items-center justify-center px-4 py-12` (mobile `py-8`).
- Логотип MPSTATS Academy — опционально, малый, по центру сверху, `text-mp-gray-400`. Не кликабельный (визард — закрытый флоу).

### Карточка-оболочка визарда (все 3 шага)
- `Card variant="default"` (белая, `shadow-mp-card`, `rounded-xl`).
- Ширина: `max-w-2xl` (672px) на шагах 1–3; на десктопе развилки — `max-w-3xl` (768px) под две карты.
- Внутри: степпер сверху → иллюстрация/заголовок → подзаголовок → контент шага → футер-навигация.
- Mobile (`<sm`): карточка занимает всю ширину минус `px-4`, `CardContent` padding `p-4`.

### Степпер (3 шага)
- 3 сегмента/точки горизонтально. Подписи: `1. Цели` · `2. Маркетплейсы` · `3. Опыт`.
- Текущий шаг — `bg-mp-blue-500` (заливка сегмента) + подпись `text-mp-blue-700 font-semibold`.
- Пройденные — `bg-mp-blue-500`, будущие — `bg-mp-gray-200`, подписи `text-mp-gray-400`.
- Можно переиспользовать паттерн полоски из `diagnostic/ProgressBar.tsx` (h-2 rounded-full), но
  сегментированный (3 равные доли). Развилка степпером НЕ нумеруется и не показывает 4-й сегмент.
- Mobile: подписи скрываются (`hidden sm:flex`), остаются только сегменты + текст «Шаг N из 3» (`text-caption`).

### Чипы целей (шаг 1, multi-select, 7 шт.)
- Раскладка: `flex flex-wrap gap-2` (или 2-колоночная grid на десктопе для длинных подписей).
- Каждый чип — кнопка-toggle: `rounded-full`, `px-4 py-2`, `text-body-sm`, иконка lucide слева (`size-4`).
- Невыбран: `border border-mp-gray-200 bg-white text-mp-gray-700 hover:border-mp-blue-300 hover:bg-mp-gray-50`.
- Выбран: `border-2 border-mp-blue-500 bg-mp-blue-50 text-mp-blue-700` + чекмарк-иконка.
- Тач-таргет `min-h-11`. Состояние focus — `focus-visible:ring-2 ring-mp-blue-500 ring-offset-2`.

### Поле свободного текста (шаг 1)
- `Textarea` (shadcn) — multi-line, ~3 строки высотой, `resize-none`. Необязательное.
- Placeholder: `Напишите, и мы поможем подобрать материалы…`
- Privacy-нота под полем: `text-caption text-mp-gray-400`.

### Карточки маркетплейсов (шаг 2, multi-select, 7 шт.)
- Grid: `grid grid-cols-2 sm:grid-cols-3 gap-3`. «Другое» может занимать оставшуюся ячейку.
- Каждая карточка — `Card`-подобная кнопка: логотип/иконка маркетплейса сверху + название (`text-body-sm`).
- Невыбрана: `border border-mp-gray-200 bg-white hover:shadow-mp-card-hover hover:-translate-y-0.5`.
- Выбрана: `border-2 border-mp-blue-500 bg-mp-blue-50` + чекмарк-иконка (`lucide CheckCircle2`,
  `text-mp-blue-600`) в правом верхнем углу.
- Высота карточек одинаковая. Тач-таргет — вся карточка, `min-h-20` (80px).

### Опции опыта (шаг 3, single-select, 4 шт.)
- Вертикальный список карточек-радио: `flex flex-col gap-3`.
- Каждая — full-width кликабельная строка: радио-индикатор слева (кружок, при выборе — `mp-blue-500`
  заливка), заголовок уровня `text-body font-semibold` + поясняющая строка `text-body-sm text-mp-gray-500`.
- Выбрана: `border-2 border-mp-blue-500 bg-mp-blue-50`. Невыбрана: `border-mp-gray-200 bg-white`.

### Футер-навигация шагов 1–3
- `flex items-center justify-between`.
- Слева: `Button variant="ghost"` «← Назад» (на шаге 1 — скрыт или disabled).
- Центр: компактный индикатор шагов (опционально, дублирует степпер на мобильном).
- Справа: `Button variant="default"` «Далее →». На шаге 1 — «Продолжить» (поле необязательно,
  кнопка всегда активна). На шагах 2–3 кнопка активна всегда (минимальный выбор не обязателен —
  визард не должен блокировать; «Другое»/пропуск допустимы).
- Helper-строка под футером: `Ответы помогут персонализировать ваш опыт.` — `text-caption text-mp-gray-400`.

### Honest reframe (шаг 1 → шаг 2)
- После сабмита шага 1 — одна строка-эхо: `text-body text-mp-gray-700` с префиксом «Поняли — хотите …».
- Клиентский шаблон, без LLM. Показывается как мягкий inline-блок (`bg-mp-blue-50 rounded-lg p-4`),
  без имитации «ответа ассистента». Может появляться кратко перед переходом или как заголовок шага 2.

### Развилка (финальный экран)
- Заголовок `text-display-sm` (mobile `text-heading-xl`), подзаголовок `text-body text-mp-gray-500`.
- Две **равноценные** карты бок о бок (`grid sm:grid-cols-2 gap-4`); на мобильном — стек, диагностика сверху.
  - **Левая — «Пройти диагностику»** (`mp-blue`): иконка-мишень/график `text-mp-blue-600` в плашке
    `bg-mp-blue-100 rounded-2xl`; 3 бейджа `Badge variant="primary"`: «15 вопросов» · «10–15 минут» ·
    «Персональные рекомендации»; CTA `Button variant="default"` «Пройти диагностику».
  - **Правая — «Перейти в обучение»** (`mp-green`): иконка-книга `text-mp-green-600` в плашке
    `bg-mp-green-100 rounded-2xl`; 3 бейджа `Badge variant="success"`: «Все курсы» · «Свой темп» ·
    «Без теста»; CTA `Button variant="success"` «Перейти в обучение».
  - Карты равной высоты: `Card` + `flex flex-col`, CTA прижата `mt-auto` (паттерн из Phase 19-02).
- Под картами — честная строка: `text-body-sm text-mp-gray-500 text-center`.
- Клик по CTA любой карты → `onboarding.complete` mutation → навигация. Во время мутации CTA показывает
  состояние загрузки (disabled + текст «Сохраняем…»), вторая карта блокируется.

### Хинт-карточка диагностики на уроке (репокат `DiagnosticGateBanner`)
- НЕ блокирующая. Размещается над плеером в `learn/[id]/page.tsx` (заменяет блокирующую ветку).
- Компактная: `Card` с `border-l-4 border-mp-blue-500 bg-mp-blue-50`, padding `p-4` (НЕ `py-12`).
- Раскладка: `flex items-start gap-3` — иконка-искра `lucide Sparkles` `text-mp-blue-600` слева,
  текст в центре, кнопка-закрытие справа.
- Заголовок `text-heading-sm`: «Пройди диагностику — соберём персональный трек».
- Подтекст `text-body-sm text-mp-gray-600` (1 строка). Inline-ссылка-CTA `Button variant="link"` size sm
  «Пройти →».
- Кнопка-закрытие: `Button variant="ghost" size="icon-sm"` с `lucide X`, `aria-label="Закрыть подсказку"`.
- После закрытия — флаг в `localStorage` (`diagnosticHintDismissed=true`), карточка не возвращается.
- Показывается ТОЛЬКО при `hasCompletedDiagnostic === false` И отсутствии флага dismissal.

## States Contract

| State | Контракт |
|-------|----------|
| Loading (`/welcome` init) | `Skeleton` карточки-оболочки или центрированный спиннер на `bg-mp-gray-50`. Гард `(main)` редиректит до рендера — на самом `/welcome` тяжёлой загрузки нет. |
| Loading (`onboarding.complete`) | CTA развилки → disabled + «Сохраняем…». Двойной клик исключён. |
| Empty (шаг 1 без выбора) | Допустимо — поле и чипы необязательны. Кнопка «Продолжить» активна. Никакого пустого-стейта-экрана нет. |
| Empty (шаг 2/3 без выбора) | Допустимо — визард не блокирует. Кнопка «Далее» активна. |
| Error (`onboarding.complete` упал) | `sonner` toast: «Не удалось сохранить ответы. Попробуйте ещё раз.» CTA возвращается в активное состояние, выбор пользователя сохранён в клиентском стейте — повтор без потери данных. |
| Success | Тихая навигация на `/diagnostic` или `/learn`. Без промежуточного success-экрана. |
| Hint dismissed | localStorage-флаг; карточка не рендерится. |

## Responsive Contract

Mobile-first обязателен (значительная доля селлеров — на телефоне).

| Breakpoint | Поведение |
|------------|-----------|
| `<sm` (<640px) | Карточка-оболочка на всю ширину минус `px-4`. Степпер — только сегменты + «Шаг N из 3». Чипы целей — `flex-wrap`. Маркетплейсы — `grid-cols-2`. Развилка — вертикальный стек, карта «Диагностика» сверху. Заголовки на ступень меньше. CTA — full-width (`w-full`). Тач-таргеты ≥44px. |

| `≥lg` | Как `sm`–`lg`; развилка `max-w-3xl`. Вертикальный ритм `py-12`. |
