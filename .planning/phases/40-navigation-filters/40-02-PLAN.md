**Fix 1: Comments email guard (R43, per D-06, D-07)**

In `packages/api/src/routers/comments.ts`, add a post-processing step to strip email from user.name. After each Prisma query that returns comments with user data, sanitize the name field.

Add a helper function at the top of the file (after imports):
```typescript
/** Strip email addresses from user name — R43 fix */
function sanitizeUserName(name: string | null): string | null {
  if (!name) return null;
  // If name looks like an email, return null (frontend shows 'Пользователь')
  if (name.includes('@') && name.includes('.')) return null;
  return name;
}
```

Then in the `list` procedure, after `const comments = await ctx.prisma.lessonComment.findMany(...)`, add:
```typescript
// Sanitize user names — never expose email (R43)
const sanitized = comments.map(c => ({
  ...c,
  user: { ...c.user, name: sanitizeUserName(c.user.name) },
  replies: c.replies.map(r => ({
    ...r,
    user: { ...r.user, name: sanitizeUserName(r.user.name) },
  })),
}));
```
Return `sanitized` instead of `comments`.

Similarly in the `create` procedure, after `const comment = await ctx.prisma.lessonComment.create(...)`:
```typescript
return {
  ...comment,
  user: { ...comment.user, name: sanitizeUserName(comment.user.name) },
};
```
