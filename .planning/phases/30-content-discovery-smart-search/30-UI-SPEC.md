## Interaction States

### Search flow
1. **Idle**: Search bar empty, example queries visible, filters visible, courses/track view active
2. **Typing**: User types query, example queries hidden, no search triggered yet
3. **Loading**: User pressed Enter, search icon replaced by spinner, "Ищем релевантные уроки..." text appears below search bar
4. **Results**: Results displayed as `SearchResultCard` list, results count shown, filters apply to results
5. **Empty results**: EmptySearchState shown with reformulation hint and popular topics
6. **Error**: Error toast via sonner + inline error message
7. **Clear**: User clicks X or clears input, returns to Idle state (courses/track view)

### Filter interactions
- **Category pills**: Single-select, click toggles, "Все" deselects others (existing pattern)
- **Status pills**: Single-select, click toggles (existing pattern)
- **Dropdowns**: Native select behavior, instant filter application
- **Topic multi-select**: Open popover, check/uncheck topics, close by clicking outside or pressing Escape
- **Marketplace pills**: Single-select, same behavior as category pills
- **Filter combination**: All filters AND together. Applied client-side via `useMemo`.
- **Filters in search mode**: Filters narrow search results client-side (don't re-trigger vector search)

### Responsive behavior
- **Desktop (md+)**: All filters in single row, cards full-width
- **Mobile (<md)**: Filters wrap to multiple rows, horizontal scroll if needed, cards stack vertically
- **Search bar**: Full width on all breakpoints
- **Example queries**: Wrap on mobile, scroll if needed

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Search input | `role="search"`, `aria-label="Поиск по урокам"` |
| Search submit | Enter key triggers search (announced via `aria-live="polite"` on results count) |
| Filter pills | `role="radiogroup"` with `role="radio"` + `aria-checked` per pill |
| Topic multi-select | shadcn Command handles `aria-multiselectable`, keyboard navigation |
| Search results | `aria-live="polite"` on results container for screen reader announcement |
| Loading state | `aria-busy="true"` on results container while searching |
| Timecode links | `aria-label="Перейти к {timecode} в уроке {lessonTitle}"` |
| Reduced motion | Existing `prefers-reduced-motion` support in globals.css |
