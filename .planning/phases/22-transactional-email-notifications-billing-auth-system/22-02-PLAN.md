   - `CQEventData` type: `Record<string, string | number | boolean | null>`

3. Create `apps/web/src/lib/carrotquest/client.ts`:
   - Class `CarrotQuestClient` with constructor taking `apiKey: string`
   - Private `request(path, body)` method: POST to `https://api.carrotquest.io/v1${path}` with `Authorization: Token ${apiKey}`, Content-Type: application/json. Log errors with `[CarrotQuest]` prefix but don't throw (fire-and-forget pattern).
   - `trackEvent(userId: string, event: CQEventName, params?: CQEventData)`: POST to `/users/${userId}/events` with `{event, params}`
   - `setUserProps(userId: string, props: Record<string, unknown>)`: POST to `/users/${userId}/props`
   - Export singleton: `export const cq = new CarrotQuestClient(process.env.CARROTQUEST_API_KEY || '')`
   - Guard: if `CARROTQUEST_API_KEY` is empty, log warning once and make all methods no-op (don't crash)

4. Create `apps/web/src/lib/carrotquest/emails.ts`:
   - Import `cq` from `./client` and `isFeatureEnabled` from `@mpstats/api/utils/feature-flags` (or direct prisma query pattern if import path issues -- use same pattern as subscription-service.ts which imports from `@mpstats/db/client`)
   - Helper `isEmailEnabled()`: checks feature flag `email_notifications_enabled`. If flag doesn't exist or is false, return false. Cache result for 60 seconds to avoid DB query on every email.
   - `sendPaymentSuccessEmail(userId, data: {amount, courseName?, periodEnd})`: guard with isEmailEnabled(), then `cq.trackEvent(userId, '$payment_success', {...})`
   - `sendPaymentFailedEmail(userId, data: {courseName?})`: similar pattern
   - `sendCancellationEmail(userId, data: {courseName?, accessUntil})`: similar
   - `sendWelcomeEmail(userId, data: {name, email})`: similar
   - `sendDiagnosticCompletedEmail(userId, data: {scores, recommendedPath?, resultsUrl})`: similar
   - All functions are async, non-throwing (wrap in try/catch, log with `[Email]` prefix)

5. Create `apps/web/src/components/ui/sonner.tsx`:
   - shadcn/ui pattern: "use client", import `Toaster as Sonner` from "sonner"
   - Style with mp-blue for action buttons, white bg, gray-950 text
   - Export `Toaster` component

6. Update `apps/web/src/app/layout.tsx`:
   - Import `Toaster` from `@/components/ui/sonner`
   - Add `<Toaster />` after `{children}` inside the body, but before closing `</body>`
   - Position: top-right (default for sonner)
