7. CQ event name для триггера (напр. `$payment_success`)

**9 писем:**

**Billing (4):**
1. Успешная оплата -- подтверждение платежа, сумма, период. Event: `$payment_success`. Vars: name, amount, course_name, period_end
2. Отказ платежа -- обновите карту / попробуйте снова. Event: `$payment_failed`. Vars: name, course_name
3. Отмена подписки -- подтверждение отмены, доступ до даты. Event: `$subscription_cancelled`. Vars: name, course_name, access_until
4. Скоро истекает -- напоминание за 3 дня до конца периода (только для CANCELLED). Event: `$subscription_expiring`. Vars: name, course_name, access_until, renew_url

**Auth/System (5):**
5. Welcome -- приветствие после регистрации, CTA: пройти диагностику. Event: `$user_registered`. Vars: name, diagnostic_url
6. Подтверждение email -- ссылка верификации. Event: через Supabase Send Email Hook (не CQ event). Vars: name, confirm_url
7. Сброс пароля -- ссылка для сброса. Event: через Supabase Send Email Hook. Vars: name, reset_url
8. Диагностика готова -- результаты + персональный трек. Event: `$diagnostic_completed`. Vars: name, scores (5 осей), recommended_path, results_url
9. Неактивность -- цепочка 3 шага (7/14/30 дней). Events: `$inactive_7d`, `$inactive_14d`, `$inactive_30d`. Vars: name, last_activity, platform_url

**Flow-схемы (текстовые, Mermaid-style):**
- Registration flow: Регистрация → Welcome → [Диагностика] → Диагностика готова
- Billing flow: Оплата → Успешная оплата → [Рекуррент] → ... → Отмена → Скоро истекает
- Inactivity flow: Нет активности 7д → 14д → 30д
