**Версия:** 1.0

**Владелец продукта:** Егор Васильев (CBDO MPSTATS)

## 1. Executive Summary

### 1.1 Проблема

Существующие образовательные курсы для селлеров маркетплейсов имеют две фундаментальные проблемы:

1. **Неэффективность обучения:** Студенты проходят 100% материала, хотя 60-80% им уже знакомо. Это демотивирует и снижает completion rate.

2. **Отсутствие персонализации:** Линейные курсы не учитывают индивидуальный уровень и пробелы в знаниях.

### 1.2 Решение

MVP платформы с двумя ключевыми модулями:

- **Diagnostic Lens:** AI-диагностика навыков на основе контента курсов
- **Adaptive Path:** Персонализированный трек обучения, скрывающий известный материал

### 1.3 Ключевые метрики успеха MVP

| Метрика | Baseline | Target (3 мес) |
|---------|----------|----------------|
| Completion rate диагностики | N/A | ≥ 70% |
| Time-to-first-lesson | N/A | < 10 мин после регистрации |
| NPS | N/A | ≥ 30 |
| Retention D7 | N/A | ≥ 40% |

## 2. Целевая аудитория

### 2.1 Primary Persona: "Растущий селлер"

```yaml
Имя: Анна
Возраст: 28-40
Опыт: 6-18 месяцев на маркетплейсах
Оборот: 300K - 2M ₽/мес
Боли:
  - "Я уже не новичок, но курсы начинаются с азов"
  - "Не понимаю, где мои слабые места"
  - "Нет времени смотреть 60 часов видео"
Цели:
  - Масштабировать бизнес до 5M+
  - Закрыть конкретные пробелы (SEO, реклама, аналитика)
  - Учиться эффективно, не тратя время на известное
Поведение:
  - Активный пользователь MPSTATS
  - Смотрит обучение с телефона в дороге
  - Ценит структуру и конкретику
```

### 2.2 Secondary Persona: "Новичок"

```yaml
Имя: Дмитрий
Возраст: 25-35
Опыт: < 3 месяцев или только планирует
Оборот: 0 - 100K ₽/мес
Боли:
  - "Не знаю с чего начать"
  - "Информации слишком много"
Цели:
  - Запустить первые продажи
  - Понять базовые принципы
Поведение:
  - Пробует бесплатные материалы
  - Нужна пошаговая навигация
```

### 2.3 Out of Scope для MVP

- Агентства (B2B)
- Опытные селлеры 10M+ (им нужен Evergreen Engine)
- Пользователи без базового понимания e-commerce

## 4. Нефункциональные требования

### 4.1 Производительность

| Требование | Значение |
|------------|----------|
| Page Load Time | < 2s (LCP) |
| API Response Time | < 500ms (p95), исключая LLM-запросы |
| LLM Response Time | < 5s (p95) |
| Concurrent Users | 100 (MVP) |

### 4.2 Безопасность

| Требование | Реализация |
|------------|------------|
| Аутентификация | JWT + httpOnly cookies |
| Пароли | bcrypt, минимум 8 символов |
| Rate limiting | 100 req/min для API, 10 req/min для LLM |
| HTTPS | Обязательно |
| Input validation | Zod schemas |

### 4.3 Доступность

| Требование | Значение |
|------------|----------|
| WCAG Level | AA (минимум) |
| Screen reader support | Базовая поддержка |
| Keyboard navigation | Полная |
| Color contrast | ≥ 4.5:1 |

### 4.4 Масштабируемость

| Аспект | MVP | Рост |
|--------|-----|------|
| Users | 1,000 | 10,000+ |
| Content | 4-7 курсов | 20+ курсов |
| RAG chunks | ~50,000 | ~200,000 |

## 5. Ограничения и допущения

### 5.1 Ограничения (Constraints)

- Биллинг НЕ входит в MVP
- Интеграция с MPSTATS Data НЕ входит в MVP
- Evergreen Engine (dynamic RAG) НЕ входит в MVP
- Геймификация (баллы, ачивки) НЕ входит в MVP

### 5.2 Допущения (Assumptions)

- Транскрипты видео будут предоставлены (Whisper pipeline)
- Видео хостятся на внешней платформе (Kinescope/YouTube)
- Дизайн-система будет предоставлена (Figma)
- Один язык интерфейса (русский)

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
